# 山海經：靈獸錄 — 遊戲規格書

> 版本 v0.4 Demo | Phaser 3 + TypeScript + Vite

---

## 一、遊戲概要

以《山海經》為世界觀的靈獸收集 RPG。玩家扮演靈獸師，在五大程序化生成的大地圖探索、捕獲靈獸、挑戰訓練師，最終擊敗冥王·幽羅。三種結局路線。

- **畫面**：640×480 像素風格 (pixelArt)，自動縮放 FIT
- **視角**：俯瞰式 tile-based 地圖探索 + 回合制戰鬥
- **平台**：瀏覽器 (Vite dev server / 靜態部署)

## 二、功法系統（開局選擇）

開局選擇一種修煉功法，決定整場遊戲的戰鬥風格。三種功法不可更換。

### 1. 御獸神訣
- 選擇一隻**本命靈寵**，該寵全能力 ×1.5
- 戰鬥方式：標準靈獸對戰（派寵出戰）
- 適合：專注培養單一強力靈寵

### 2. 萬靈化型變
- 玩家可**變身**為任何「見過的」靈獸出戰，全能力 ×1.1
- HP% 繼承：變身時按目前血量百分比換算（例：70% → 新形態的 70%）
- 可隨時切換**變身形態**或**派出包包靈寵**
- 變身被打倒時，若包包有活寵可繼續戰鬥
- 適合：靈活切換、百變戰術

### 3. 煉天大法
- 人類**親自上場戰鬥**，與靈寵形成雙實體
- 人類技能：輕擊(40/100%)、重擊(90/70%)、防禦(減傷50%)、煉化
- **煉化**：敵方 HP < 15% 時可嘗試吸收，成功率 = 50% + (玩家等級-敵等級)×2% + (15-敵HP%)×1%
- 煉化成功：吸收敵方全屬性 5%，永久加成人類數值
- 可煉化靈獸與人類（死鬥中）
- 適合：肉搏型玩家

## 三、戰鬥系統

### 3.1 普通戰鬥（野生 / 訓練師）
- 回合制：速度高者先攻
- 每回合選擇：技能 / 換獸 / 捕獲(野生) / 逃跑
- 技能有 PP 限制，用完不可使用
- 經驗值分配：擊敗靈獸 → 參戰靈寵獲 EXP，人類獲 50% EXP

### 3.2 死鬥模式（訓練師對話選擇）
- **順序制**：先打靈寵 → 靈寵全滅 → 人類上場
- 人類被打倒 = **GAME OVER**（回到主選單）
- 勝利：奪取對方全部靈寵，NPC 從地圖消失
- 失敗：完全重來
- 死鬥可選擇「認輸」棄權撤退（除 BOSS 戰外）

### 3.3 傷害公式
```
基礎傷害 = ((2×等級/5+2) × 技能威力 × (攻擊/防禦)) / 50 + 2
最終傷害 = 基礎傷害 × 屬性相剋 × 同屬加成(1.3) × 暴擊(1.5) × 隨機(0.85~1.0) × 階級修正
```

### 3.4 階級傷害修正（新增）
- 基於修仙境界差異，每差一個大階段 ±10%
- 練氣打築基：傷害 ×0.9，築基打練氣：傷害 ×1.1
- 差兩階：±20%，依此類推

### 3.5 越階經驗加成
- 擊敗比自己高階的敵人，每高一個境界 +75% 經驗
- 差兩階 = +150%，差三階 = +225%

### 3.6 經驗公式
- 擊敗靈獸基礎 EXP：`等級 × 25 + 30`
- 同屬性練化（吸收）：EXP ×1.25
- 同角色練化（吸收）：EXP ×1.5
- 設計目標：每張地圖停留約 10 分鐘

## 四、五行屬性系統

```
金 → 克木 → 克土 → 克水 → 克火 → 克金（循環）
```

| 攻擊屬性 | 剋制 | 被剋 |
|---------|------|------|
| 金      | 木   | 火   |
| 木      | 土   | 金   |
| 土      | 水   | 木   |
| 水      | 火   | 土   |
| 火      | 金   | 水   |

- 剋制：傷害 ×1.5
- 被剋：傷害 ×0.67
- 同屬攻擊 (STAB)：傷害 ×1.3

## 七、地圖與 NPC

### 7.1 五大地圖（程序化生成）

地圖使用 seeded PRNG (Mulberry32, seed=42) 程序化生成，確保每次一致。
渲染使用 viewport culling（tile pool ~300 個重複使用 Image），避免大地圖效能爆炸。

## 八、練妖壺（融合 + 吸收）

### 8.1 融合（練妖壺）
- 從背包選單（B 鍵）直接使用，無需找 NPC
- 選擇兩隻靈獸（主體 + 素材），素材被消耗
- 融合結果：主體獲得素材 20% 基礎屬性加成
- **異色機率**：20% 機率出現異色個體（額外 +10% 全屬性）
- 御獸神訣：本命靈寵融合後失去本命標記

### 8.2 吸收（練化互吃）
- 從背包選單直接使用
- 犧牲一隻靈獸，將其累積 EXP 餵給另一隻
- **同屬性加成**：吸收同屬性靈獸，EXP ×1.25
- **同角色加成**：吸收相同靈獸，EXP ×1.5

## 九、人類戰鬥屬性

- 基礎公式：`avg(隊伍屬性) × 0.3 × (1 + (等級-1)×0.03) + 煉化加成`
- 初始等級：練氣5等
- 經驗來源：戰鬥勝利獲得靈獸 EXP 的 50%
- 最高等級：42（與靈獸相同）
- 死鬥中人類技能：輕擊 / 重擊 / 防禦 / 撤退

## 十一、場景流程

```
Boot (載入資源)
  → Intro (開場動畫 + 劇情)
    → MainMenu (功法選擇 / 繼續遊戲)
      → Overworld (地圖探索)
        ↔ Battle (戰鬥)
        → Ending (三種結局)
```

### 三種結局

| 結局        | 條件                                   | 畫面主題         |
|------------|----------------------------------------|-----------------|
| **True End**  | 不靠煉化靈寵，正面擊敗 BOSS 人類         | 金色星空，歌頌羈絆 |
| **Normal End** | BOSS 人類階段選擇「煉化全寵」後獲勝      | 暗棕色，孤獨的勝利 |
| **Bad End**    | 在 BOSS 戰中被打倒（人類 HP 歸零）       | 暗紅色，沉淪冥界   |

- **Normal End 路線**：BOSS 人類階段時可選「煉化全寵」，犧牲全部靈寵獲得其 80% 總屬性 + 滿血回復
- **Bad End**：直接進入結局畫面（非主選單），有專屬敘事

## 十三、技術架構

### 程序化地圖生成（mapGenerator.ts）
- **PRNG**：Mulberry32 (seed=42)，確保每次生成相同地圖
- **地形**：Value noise 2D + fractal octave stacking
- **道路**：加權隨機行走 (carvePath) 打通 waypoint 連接
- **驗證**：BFS flood fill 確保所有重要點可達
- **NPC 放置**：findNearestWalkable() 在 waypoint 附近找可行走格

### 渲染優化（OverworldScene.ts）
- **Tile Pool**：~300 個可重複使用 Image，避免 128×128=16384 個物件
- **Viewport Culling**：每幀計算可見範圍，只渲染 ~200 格
- **Camera Bounds**：攝影機限制在地圖範圍內
